git push -> Envia os commits(alteracoes salvas) do repositorio local do seu pc para o repositório remoto do github;

------------------------------------------------------------------------------------------------------------------------------------------
git pull

Server para recebermos atualizações do repositorio remoto;

Verifica se tem arquivos modificados no diretorio remoto do GitHub. Se tiver, ele puxa para o repositorio local; 

Ele dá um git fetch e um git merge;

Você pode dar um "git pull" em cada branch; 

Numa empresa, os devs trabalham junto com 1 ou varios repositorios; Não é só voce que vai dar um "git push" e mandar arquivos para o repositorio remoto GitHub. Os outros devs também vao enviar.


Então... De tempos em tempos é interessante dar um "git pull" pra atualizar os arquivos do repositorio local e deixar igualzin o do repositorio remoto GitHub;



1- Vá no repositorio remoto GitHub e crie 1 arquivo. Crie de forma manual mesmo(clicando no GitHub);

Beleza. Imagine que quem criou esse arquivo foi outro dev da sua equipe.


2- Atualize o repositorio remoto GitHub e atualize seu repositorio local. 
        git pull;


Viu?? O arquivo foi puxado para o seu repositorio local;



3- Vá no GitHub novamente e edite esse arquivo;

4- Dẽ um git pull e puxe os arquivos do repositorio remoto para o repositorio local.
        git pull origin master;  ou
        git pull origin main;

Se ligou??


Resumindo:

git push -> Empurra os arquivos commitados do repositorio local para o repositorio remoto do GitHub;
git pull -> Puxa os arquivos commitados do repositorio remoto do GitHub para o repositorio local;

git push-> Empurra para o GitHub;
git pull-> Puxa para o repositorio local;




Como boa prática, sempre é bom dar um "git pull" antes de criar uma branch;
Assim, garantimos que a branch foi criada com a ultima versao da master;
